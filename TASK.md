# Тестовое задание: дайджест новостей по запросу на Hermes Agent

**Уровень:** Junior AI Developer
**Ориентир по времени:** 8–12 часов (1–2 вечера)
**Рекомендуемый дедлайн:** 3–4 дня
**Формат сдачи:** ссылка на git-репозиторий или архив с проектом

---

## 1. Легенда

Нужен агент, который по **запросу из Telegram-чата** собирает дайджест новостей
**за указанный период** по заданной теме, пишет связную статью на русском и присылает её
обратно в тот же чат.

Пример диалога:

> **Пользователь:** собери дайджест по теме «AI / ML» за последнюю неделю
> **Бот:** *(вызывает инструменты, обрабатывает, отвечает готовой статьёй-дайджестом)*

Ключевая идея: ты **не пишешь свой LLM-цикл руками** — ты оркеструешь готового
автономного агента (**Hermes Agent**) и даёшь ему **свой инструмент** (MCP-сервер)
для доступа к источнику новостей. Логику «как писать дайджест» оформляешь как **skill**.

Логическая схема, которую нужно собрать:

```
Telegram (запрос пользователя) ──► Hermes Agent ──► skill: news-digest (КАК собрать статью)
                                        │
                                        ├──► MCP: mcp-news (ОТКУДА брать данные за период)
                                        │
                                        └──► Telegram (ответ — готовая статья)

всё запакетировано в docker compose: hermes-agent + mcp-news,
общая сеть, volume для памяти агента
```

Разделение ответственности (важно понимать и уметь защитить):
- **MCP** = «руки»: доступ к данным за период (`get_news` / `search_habr` / `fetch_article`).
- **Skill** = «привычка»: процедура, *как* из данных собрать статью.
- **Telegram** = вход (запрос) и выход (ответ).

## 2. Что мы проверяем

- Умение поднять и сконфигурировать готовый агентный фреймворк.
- Понимание **MCP**: как написать свой сервер с инструментами и подключить его к агенту.
- Понимание **skills** Hermes: вынести процедуру в переиспользуемый навык, а не в разовый промпт.
- Умение агента **разобрать запрос на естественном языке** (тема + период) и вызвать инструменты с нужными параметрами.
- Грамотный **промптинг** под задачу (структура статьи, тон, дедупликация, фактологичность).
- **Docker**: воспроизводимость, сеть между сервисами, секреты, персистентность.
- Базовая инженерия: env/секреты, README, обработка ошибок сети.

## 3. Стек (обязательно)

- **Hermes Agent** как раннер агента — https://github.com/nousresearch/hermes-agent
  (LLM на выбор: бесплатная через OpenRouter, локальная и т.п. — указать, какую брал).

- **Docker + docker compose** — всё поднимается одной командой.

## 4. Уровни

Выбрать **минимум Level 1**. Level 2 и 3 повышают оценку.

### Level 1 — База (обязательно)
MCP-инструмент `get_news(topic, period, limit)` тянет материалы за период из **RSS**
(Habr RSS по хабам или любой новостной RSS). По запросу из Telegram агент разбирает
тему и период, вызывает инструмент, пишет дайджест из 5–7 пунктов
(заголовок + 2–3 предложения сути + ссылка) и отвечает в чат.

### Level 2 — Хабр-вариант (рекомендуется как основной)
Инструмент `search_habr(query, period)` ищет статьи на Хабре за период,
инструмент `fetch_article(url)` достаёт текст. Агент **аннотирует** каждую статью
(TL;DR на 2–3 строки) и собирает из аннотаций **цельную обзорную статью** с вводкой
и выводом — не просто список.

### Level 3 — Память и качество (бонус)
Агент использует **память Hermes**, чтобы между запросами не повторять материалы,
которые уже попадали в прошлые дайджесты (дедупликация). Опционально — генерация
обложки-картинки к дайджесту.

## 5. Контракт MCP-сервера

| Tool | Вход | Выход |
|---|---|---|
| `get_news` / `search_habr` | `topic/query: str`, `period: str`, `limit: int` | список `{title, url, published_at, source, snippet}` |
| `fetch_article` *(L2+)* | `url: str` | `{title, text, author, published_at}` |

Требования:
- `period` — за какой срок брать материалы (напр. `"7d"`, `"24h"`, или диапазон дат);
  фильтрация по дате выполняется на стороне MCP.
- Валидные JSON-схемы инструментов и **понятные описания** — агент по описанию должен
  понять, когда какой инструмент звать и как заполнить `period` из запроса пользователя.
- Таймауты и аккуратная обработка пустого ответа / 404 / падения источника.
- Никакого хардкода секретов.

---

## 6. Требования к Skill

Логику дайджеста оформить как **кастомный skill** `news-digest`, а не как разовый
промпт. Skill должен описывать процедуру:

1. как извлечь из запроса пользователя **тему и период**;
2. какие инструменты MCP звать и в каком порядке;
3. как отбирать и **дедуплицировать** материалы;
4. структуру статьи (вводка → пункты → вывод);
5. тон и язык (русский, нейтрально-технический);
6. формат ответа в Telegram.

---

## 7. Требования к Docker

Всё решение в идеале поднимается **одной командой** (если все-таки требуется несколько команд,
указать в документации обязательно), без ручной установки зависимостей на хост.

- `Dockerfile` для **MCP-сервера**.
- `docker-compose.yml` поднимает **два сервиса**: `hermes-agent` и `mcp-news`,
  в одной сети, чтобы Hermes видел MCP **по имени сервиса** (не по `localhost`).
- Секреты и токены — **только через env / `.env`** (проброс в compose),
  ничего не зашито в образ.
- Персистентность: память и конфиг Hermes — в **volume**, чтобы дедуп (L3)
  и состояние переживали рестарт.
- Запуск:
  ```
  cp .env.example .env   # вписать токены
  docker compose up
  ```
